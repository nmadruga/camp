Chemistry Across Multiple Phases
CAMP Documentation

Chemistry Across Multiple Phases (CAMP) facilitates multi-phase chemistry solving in atmospheric models. In general, CAMP solves a mechanism composed of a set of reactions over a time-step specified by the host model. Reactions can take place in the gas phase, in one of several aerosol phases, or across an interface between phases (gas–aerosol). CAMP is designed to work with any aerosol representation used by the host model (e.g., binned, modal, single particle) through the use use of custom extending types of the abstract camp_aero_rep_data::aero_rep_data_t type. Each aerosol representation implements one or more instances of each aerosol phases based on the configuration of the host model. The multi-phase chemistry system is solved for the gas-phase and every instance of each aerosol phase. A set of sub-models may also be included to calculate parameters needed by reactions during solving.

CAMP uses JSON files to load chemical species, mechanisms, aerosol phases, aerosol representations, and sub-models at runtime. This allows a user to modify any of these data without recompiling the model, permits host models to choose which mechanisms to solve based on model conditions, and allows multiple mechanisms to be solved during a model simulation.

Usage

Input files

CAMP uses two types of input files:

  • File List A json file containing a list of CAMP configuration file names.
  • Configuration File One or more json files containing all the CAMP configuration data.

To initialize CAMP , the path to the file list must be passed to the camp_camp_core::camp_core_t constructor. The method by which this is done depends on the host model configuration.

Adding CAMP to an atmospheric model

The easiest way to incorporate CAMP into an atmospheric model is to follow the templates laid out in the tests. For each reaction, a test exists in test/unit_rxn_data that initializes CAMP for a simple mechanism consisting of only the reaction type being tested. The test code is documented so that it can be used as a template for incorporating CAMP in a new host model.
