\mbox{\hyperlink{index}{CAMP}} is software for solving multi-\/phase chemistry in atmospheric models. For a overview of what \mbox{\hyperlink{index}{CAMP}} can do, check out \mbox{\hyperlink{camp_tutorial_part_0}{part 0 of the CAMP tutorial}}. \mbox{\hyperlink{index}{CAMP}} uses json input files to load \mbox{\hyperlink{input_format_species}{chemical species}}, \mbox{\hyperlink{input_format_mechanism}{mechanisms}}, \mbox{\hyperlink{input_format_aero_phase}{aerosol phases}}, \mbox{\hyperlink{input_format_aero_rep}{aerosol representations}}, and \mbox{\hyperlink{input_format_sub_model}{sub-\/models}} at runtime. You can also check out ./camp/\+Dockerfile (or ./camp/\+Dockerfile.mpi for MPI applications) to see how CAMP is is built for automated testing.\hypertarget{index_autotoc_md9}{}\doxysubsection{Input files}\label{index_autotoc_md9}
\mbox{\hyperlink{index}{CAMP}} uses two types of input files\+:


\begin{DoxyItemize}
\item \mbox{\hyperlink{input_format_camp_file_list}{File List}} A {\ttfamily json} file containing a list of \mbox{\hyperlink{index}{CAMP}} configuration file names.
\item \mbox{\hyperlink{input_format_camp_config}{Configuration File}} One or more {\ttfamily json} files containing all the \mbox{\hyperlink{index}{CAMP}} configuration data.
\end{DoxyItemize}

To initialize \mbox{\hyperlink{index}{CAMP}} , the path to the \mbox{\hyperlink{input_format_camp_file_list}{file list}} must be passed to the \mbox{\hyperlink{structcamp__camp__core_1_1camp__core__t}{camp\+\_\+camp\+\_\+core\+::camp\+\_\+core\+\_\+t}} constructor. The method by which this is done depends on the host model configuration.\hypertarget{index_autotoc_md10}{}\doxysubsection{CAMP tutorial}\label{index_autotoc_md10}
